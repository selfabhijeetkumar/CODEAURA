## ⚙️ How It Actually Works — Under The Hood

<br/>

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  THE CODEAURA EXECUTION PIPELINE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  STEP 01 ── INPUT
  ─────────────────────────────────────────────────────────────────────
  User pastes any code (Python, JavaScript, Java, C++, etc.)
  into the Monaco Editor — the same editor that powers VS Code.

                              ↓

  STEP 02 ── AI ANALYSIS
  ─────────────────────────────────────────────────────────────────────
  Frontend sends code to backend  →  POST /api/v1/analyze

  Google Gemini 1.5 Flash receives the code with a carefully
  engineered system prompt that instructs it to:

    •  Trace every execution step sequentially
    •  Identify the step TYPE  (function_call, loop, conditional…)
    •  Write a plain-English NARRATION for each step
    •  Generate a CINEMATIC HINT for visual direction
    •  Extract PAYLOAD data  (variable names, values, depth…)
    •  Return a structured JSON array of 15–20 execution steps

  If Gemini fails  →  OpenAI GPT-3.5 Turbo activates as fallback.
  Zero downtime. Always works.

                              ↓

  STEP 03 ── STRUCTURED EXECUTION MAP
  ─────────────────────────────────────────────────────────────────────
  Returns clean JSON:

  [
    {
      "step": 1,
      "type": "function_call",
      "title": "Entering factorial()",
      "narration": "We call factorial with n=5. A new stack frame opens...",
      "cinematicHint": "PORTAL_OPEN_DEEP",
      "payload": { "name": "factorial", "args": { "n": 5 } }
    },
    {
      "step": 2,
      "type": "conditional",
      "title": "Base Case Check",
      "narration": "Checking if n equals zero. It does not — n is 5.",
      "cinematicHint": "GATE_RIGHT_OPENS",
      "payload": { "condition": "n === 0", "result": false }
    }
  ]

                              ↓

  STEP 04 ── THE CHOREOGRAPHER
  ─────────────────────────────────────────────────────────────────────
  The Choreographer reads each step's TYPE and maps it to its actor:

    "function_call"  →  <FunctionPortal  />
    "loop_start"     →  <LoopHelix       />
    "conditional"    →  <ConditionalGate />
    "error"          →  <ErrorShatter    />
    … and 10+ more

                              ↓

  STEP 05 ── CINEMATIC RENDERING
  ─────────────────────────────────────────────────────────────────────
  Three.js + React Three Fiber renders the actor in full 3D.
  GSAP fires cinematic entrance and exit animations.
  Camera moves. Lights shift. The scene breathes.

                              ↓

  STEP 06 ── NARRATION
  ─────────────────────────────────────────────────────────────────────
  Web Speech API reads the step's narration aloud — synchronized
  perfectly with the 3D animation. No delay. No mismatch.

                              ↓

  STEP 07 ── UNDERSTANDING
  ─────────────────────────────────────────────────────────────────────
  The user watches. Hears. Sees. Understands.
  For the first time. Truly. Completely.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 🗃️ Database Schema — Full Blueprint

<br/>

```sql
-- ═══════════════════════════════════════════════════════════════════════
--                       CODEAURA DATABASE SCHEMA
--                    Supabase PostgreSQL — Full Blueprint
-- ═══════════════════════════════════════════════════════════════════════


-- ───────────────────────────────────────────────────────────────────────
-- TABLE 01: sessions
-- Stores every code analysis session a user runs.
-- The core data entity of CODEAURA.
-- ───────────────────────────────────────────────────────────────────────

CREATE TABLE sessions (
  id                uuid          PRIMARY KEY DEFAULT gen_random_uuid(),
  session_name      text          NOT NULL,
  code              text          NOT NULL,
  language          text,
  steps             jsonb,
  -- Full execution step array returned from Gemini AI
  quality_score     integer,
  -- Overall AI quality rating: 0–100
  quiz_score        integer,
  -- User's performance score on the generated quiz
  bug_count         integer,
  -- Total number of bugs detected by AI analysis
  complexity_time   text,
  -- Big-O time complexity, e.g. "O(n log n)"
  complexity_space  text,
  -- Big-O space complexity, e.g. "O(n)"
  share_token       uuid          DEFAULT gen_random_uuid(),
  -- Unique token used to generate public share links
  created_at        timestamptz   DEFAULT now(),
  updated_at        timestamptz   DEFAULT now()
);

-- Index for fast share link token lookups
CREATE INDEX idx_sessions_share_token ON sessions(share_token);

-- Index for chronological session history
CREATE INDEX idx_sessions_created_at ON sessions(created_at DESC);


-- ───────────────────────────────────────────────────────────────────────
-- TABLE 02: session_notes
-- User annotations on individual execution steps.
-- Powers the "learn by writing" feature.
-- ───────────────────────────────────────────────────────────────────────

CREATE TABLE session_notes (
  id                uuid          PRIMARY KEY DEFAULT gen_random_uuid(),
  session_id        uuid          REFERENCES sessions(id) ON DELETE CASCADE,
  step_number       integer       NOT NULL,
  ai_explanation    text,
  -- AI's deep-dive explanation generated for this specific step
  user_note         text,
  -- User's own personal annotation or reflection
  created_at        timestamptz   DEFAULT now()
);

-- Index for fast per-session note retrieval
CREATE INDEX idx_notes_session_id ON session_notes(session_id);

-- Unique constraint: one note entry per step per session
CREATE UNIQUE INDEX idx_notes_session_step
  ON session_notes(session_id, step_number);


-- ───────────────────────────────────────────────────────────────────────
-- TABLE 03: leaderboard_stats
-- Global competition rankings based on code quality scores.
-- ───────────────────────────────────────────────────────────────────────

CREATE TABLE leaderboard_stats (
  id                uuid          PRIMARY KEY DEFAULT gen_random_uuid(),
  username          text          NOT NULL UNIQUE,
  total_sessions    integer       DEFAULT 0,
  average_score     float         DEFAULT 0.0,
  best_score        integer       DEFAULT 0,
  total_bugs_fixed  integer       DEFAULT 0,
  rank_badge        text          DEFAULT 'Getting Started',
  -- Badge tiers:
  --   'Getting Started'   →  average  0 – 40
  --   'Rising Coder'      →  average 41 – 60
  --   'Sharp Developer'   →  average 61 – 75
  --   'Elite Programmer'  →  average 76 – 90
  --   'Code Overlord'     →  average 91 – 100
  last_active       timestamptz   DEFAULT now(),
  created_at        timestamptz   DEFAULT now()
);

-- Indexes for efficient leaderboard ranking queries
CREATE INDEX idx_leaderboard_avg_score
  ON leaderboard_stats(average_score DESC);

CREATE INDEX idx_leaderboard_best_score
  ON leaderboard_stats(best_score DESC);
```

## 🚀 Getting Started — Run In Minutes

<br/>

### Prerequisites

```
  ✅  Node.js v18 or higher       →   node --version
  ✅  pnpm package manager        →   npm install -g pnpm
  ✅  Git                         →   git --version
  ✅  Google Gemini API Key       →   aistudio.google.com  (free tier available)
  ✅  Supabase Project            →   supabase.com         (free tier available)
  ✅  OpenAI API Key              →   platform.openai.com  (optional, fallback only)
```

<br/>

### Step 1 — Clone the Repository

```bash
git clone https://github.com/selfabhijeetkumar/CODEAURA.git
cd CODEAURA
```

<br/>

### Step 2 — Install All Dependencies

```bash
# Install frontend dependencies
cd FRONTEND
pnpm install

# Install backend dependencies
cd ../BACKEND
pnpm install
```

<br/>

### Step 3 — Configure Environment Variables

**Frontend** → Create `FRONTEND/.env.local`:

```env
# ──────────────────────────────────────────────────────────────
# CODEAURA — Frontend Environment Variables
# ──────────────────────────────────────────────────────────────

# Backend API base URL (local development)
NEXT_PUBLIC_API_URL=http://localhost:8080

# Supabase project credentials
NEXT_PUBLIC_SUPABASE_URL=https://xxxxxxxxxxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key_here

# Public app URL used for generating share links
NEXT_PUBLIC_APP_URL=http://localhost:3000
```

**Backend** → Create `BACKEND/.env`:

```env
# ──────────────────────────────────────────────────────────────
# CODEAURA — Backend Environment Variables
# ──────────────────────────────────────────────────────────────

# Server configuration
PORT=8080
NODE_ENV=development

# AI provider credentials
GEMINI_API_KEY=your_gemini_api_key_here
OPENAI_API_KEY=your_openai_api_key_here

# Supabase credentials (use service role key for backend)
SUPABASE_URL=https://xxxxxxxxxxxx.supabase.co
SUPABASE_SERVICE_KEY=your_supabase_service_role_key_here

# CORS allowed origin
CORS_ORIGIN=http://localhost:3000

# Rate limiting configuration
RATE_LIMIT_WINDOW_MS=900000
RATE_LIMIT_MAX=100
```

<br/>

### Step 4 — Initialize the Database

```
  1. Open your Supabase project dashboard at supabase.com
  2. Navigate to:  SQL Editor
  3. Open the file:  DATABASE/schema.sql
  4. Paste the full contents and click  Run
  5. All three tables will be created automatically  ✅
```

<br/>

### Step 5 — Launch CODEAURA

```bash
# Terminal 1 — Start the backend server
cd BACKEND
pnpm dev
# ✅  Backend running at http://localhost:8080

# Terminal 2 — Start the frontend
cd FRONTEND
pnpm dev
# ✅  Frontend running at http://localhost:3000
```
